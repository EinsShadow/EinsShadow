## 💫 Über mich

Ich bin ein leidenschaftlicher **Java-Entwickler** mit Spezialisierung auf die Erstellung von hochperformanten, skalierbaren und modularen **Minecraft Server-Plugins (Paper & Spigot)**. Mein Fokus liegt darauf, das Spielerlebnis durch sauberen Code zu optimieren und komplexe Datenstrukturen effizient im Hintergrund zu verwalten.

- 🛠️ Ich entwickle maßgeschneiderte Systeme für Minecraft-Netzwerke.
- 💾 Ich arbeite intensiv mit SQL- und NoSQL-Datenbanken für persistente Spielerspeicherungen.
- ⚡ Performance, Asynchronität und Cache-Optimierung stehen bei mir an erster Stelle.
- 💬 Frag mich gerne nach Plugin-Architektur, Performance-Tuning oder Spigot-API-Tweaks!

## 🎮 Minecraft Development Focus

Als Entwickler im Minecraft-Bereich weiß ich, wie wichtig Effizienz für ein lagfreies Spielerlebnis ist. Meine Plugins zeichnen sich aus durch:

* **Echtzeit-Datenverwaltung**: Nutzung von **Redis** für ultraschnelles Caching und inter-server Kommunikation (Publish/Subscribe) in BungeeCord/Velocity-Netzwerken.
* **Skalierbare Storage-Lösungen**: Daten-Backends mit **MongoDB** (für flexible Spielerdokumente) oder **MariaDB/MySQL** (für relationale Statistiken).
* **Ausfallsicherheit (Failover)**: Lokales Caching via **SQLite** oder In-Memory bei Datenbankverbindungsabbrüchen.
* **Tick-Schonung**: Weitestgehende Vermeidung von blockierendem Code auf dem Haupt-Thread durch die konsequente Nutzung asynchroner Tasks.
